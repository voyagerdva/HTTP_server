ДОПОЛНЕНИЕ №2 к ТЗ на HTTP-сервер:

1)	в классе BaseDao создать метод "connectDB" - он будет производить соединение с заранее созданной БД. Для работы с БД должен использоваться драйвер Postgresql. Имя БД задается литералом в строке оператора. Формат записи оператора должен быть таким:

 postgresql.open('pq://postgres:postgres@localhost:5432/neurodb')

2) 	в методе insert класса UserDao создать оператор, выполняющий создание новой записи в таблицу "Users", причем оператор должен состроять из трех частей:
	= часть, устанавливающая соединение с нужной БД. Т.е. это объект, возвращенный методом "connectDB" класса "BaseDao". Формат записи такой:
		BaseDao.connectDB(self).
	
	= часть PREPARE, выполняющая подготовку отложенного запроса оператора INSERT с явно указанным именем таблицы Users, явно указанными именами 5-ти полей - id, firstname, lastname, age, address и, заданными параметрически, значениями, которые требуется вписать в каждое поле. 
	Формат записи такой:
		prepare("INSERT INTO users (id, firstname, lastName, age, address) VALUES ($1, $2, $3, $4, $5)")

	= часть, исполняющая подготовленный запрос. В ней указываются конкретные значения, добавляемые в поля таблицы. Значения полей берутся из значений ключей DICT-объекта BODY. Имена полей соответствуют именам ключей BODY. Формат записи такой:
	 ins(body['id'], body['firstname'], body['lastname'], body['age'], body['address'])
	 где ins - имя объекта, устанавливающего соединение с БД и подготавливающего отложенный запрос на добавление записи в таблицу.
	 
4) 	Правильность выполнения метода INSERT проверяется фактическим рассмотрением таблицы Users в pgAdmin с помощью запроса SELECT * . Пока так.
	
5) метод insert класса UserDao должен возвращать в MyController сериализованный объект класса UserDto, который должен передаваться в метод POST сервера и отправляться в качестве ответа сервера к клиенту.
